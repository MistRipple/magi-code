# Bug 优化计划文档

> 目标：解决对话展示与编排流程的 6 类问题，保证时间轴稳定、状态与徽章/时间正确、意图分级可控、上下文与画像传递准确、多轮对话稳定、编排流程可靠。

## 一、问题范围与目标

1. 对话面板消息不能覆盖前面消息，应按时间轴向下追加展示。
2. 用户发送消息后，“正在思考”动画触发时机，以及徽章/时间展示是否正确（避免空徽章/空时间）。
3. 用户内容分级处理：何时直接回答，何时需要补充需求，何时进入拆解确认（面向画像系统）。
4. 用户画像 + 插件上下文/记忆传递是否正确。
5. 多轮、连续对话是否正常。
6. 编排流程是否正常。

## 二、现状关键点（简要）

- 消息列表按时间排序渲染；流式更新在同一条消息内增量更新。
- 编排者 progress 消息有合并逻辑，可能造成“覆盖感”。
- Thread 面板底部思考动画依据 isProcessing + 流式内容/编排者回应条件展示。
- 澄清/Worker 提问已作为普通消息渲染，并附徽章。
- 画像与上下文注入链路完整，但会话保存处有时间戳覆盖风险。

## 三、执行计划（阶段化）

### Phase 1：时间轴与覆盖问题
- 目标：确保消息稳定追加，不因新消息覆盖旧消息。
- 动作：
  - 审计进度消息合并逻辑，决定是否移除/降低合并力度。
  - 统一 message.timestamp 的来源，保证排序稳定。
  - 校验流式更新只影响当前流式消息，不影响历史消息。

### Phase 2：思考动画 + 徽章/时间渲染
- 目标：思考动画触发稳定，徽章/时间完整且不空。
- 动作：
  - 校验“正在思考/正在输出”逻辑触发点。
  - 统一徽章生成规则，避免空徽章占位。
  - 兜底 timestamp/time，避免空时间显示。

### Phase 3：意图分级策略（画像系统适配）
- 目标：明确“回答/澄清/拆解确认”边界。
- 动作：
  - 调整 IntentGate 阈值与规则，增强需求不清晰时的澄清触发。
  - 增加“复杂任务需拆解确认”的判定规则。
  - 在编排流程中对高风险任务强制确认。

### Phase 4：画像 + 上下文/记忆传递
- 目标：画像、上下文、记忆在 orchestrator/worker/CLI 三层一致。
- 动作：
  - 核对画像加载与注入的触发时机。
  - 校验 contextSnapshot 的传递与截断策略。
  - 修复会话保存时重写时间戳导致的排序漂移。

### Phase 5：多轮对话与编排流程校验
- 目标：多会话隔离、编排流转完整。
- 动作：
  - 校验 sessionId 与消息路由一致性。
  - 校验打断/恢复/确认流程的状态更新。
  - 形成最小回归用例。

## 四、任务清单（可执行）

1. 取消或调整 progress 消息合并策略，确保时间轴追加展示。
2. 统一消息时间戳来源，避免排序漂移。
3. 校准 Thread 面板“正在思考”显示条件与更新节奏。
4. 为徽章与时间字段增加兜底逻辑，避免空显示。
5. 强化意图门控规则：需求不清晰走澄清，复杂任务走拆解确认。
6. 校验画像/上下文注入链路，修复会话保存中的时间戳覆盖。
7. 增强多轮会话隔离校验，补充回归场景。

## 五、推进方式

- 按 Phase 顺序逐步落地，完成一阶段再进入下一阶段。
- 每阶段提交：变更列表 + 风险点 + 验证步骤。
