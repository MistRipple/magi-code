## 项目概述

### 目标

将 TaskStateManager 的功能合并到 UnifiedTaskManager，消除双重状态管理，解决状态不一致风险。

### 动机

**问题**:
- 系统中存在两个状态管理器：TaskStateManager 和 UnifiedTaskManager
- 双重状态管理导致状态不一致风险
- 状态映射逻辑复杂且容易出错
- 数据冗余，维护成本高

**解决方案**:
- 扩展 UnifiedTaskManager 功能
- 迁移所有使用 TaskStateManager 的代码
- 删除状态映射逻辑
- 统一类型系统

## 最终架构

### 统一的状态管理系统

```
Application Layer
  ├── OrchestratorAgent (任务编排)
  └── RecoveryHandler (失败恢复)
           │
           ▼
Task Management Layer
  ├── UnifiedTaskManager (统一任务管理)
  │   ├── 任务生命周期管理
  │   ├── 优先级调度
  │   ├── 超时管理
  │   ├── 重试机制
  │   └── 事件驱动
  │
  └── SessionManagerTaskRepository (持久化适配器)
           │
           ▼
Persistence Layer
  └── UnifiedSessionManager (会话管理)
```

### 关键特性

1. **单一状态源**: UnifiedTaskManager 是唯一的状态管理器
2. **统一类型**: 所有地方使用 SubTask 类型
3. **事件驱动**: 20+ 个细粒度事件
4. **完整 API**: 20+ 个方法覆盖所有场景
5. **类型安全**: TypeScript 类型检查保证正确性
