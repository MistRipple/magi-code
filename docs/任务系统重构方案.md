# 任务系统重构方案 - 统一状态管理

**方案类型**: 长期重构（破坏性变更）
**目标**: 彻底统一状态管理，消除所有兼容性处理
**原则**: 不允许任何向后兼容代码

---

## 一、重构目标

### 1.1 核心目标

1. **统一状态管理**: 合并 TaskManager 和 TaskStateManager 为单一系统
2. **统一类型定义**: 消除所有类型冲突和重复定义
3. **消除兼容性代码**: 删除所有 `assignedCli`、`WorkerType` 等向后兼容字段
4. **清晰的职责分离**: 明确每个组件的职责边界
5. **完整的功能支持**: 重试、暂停/恢复、优先级、超时

### 1.2 非目标

- ❌ 保持向后兼容
- ❌ 渐进式迁移
- ❌ 支持旧版 API

---

### 2.2 核心组件

#### TaskManager (统一管理器)

**职责**:
- Task/SubTask 的创建、读取、更新、删除
- 状态转换和验证
- 完成检测和自动状态更新
- 重试机制
- 优先级队列管理
- 超时检测和处理
- 事件发布

**不负责**:
- 持久化实现（委托给 TaskRepository）
- Worker 管理（委托给 WorkerPool）
- 任务编排（委托给 OrchestratorAgent）

#### TaskRepository (持久化层)

**职责**:
- Task/SubTask 数据的持久化存储
- 查询接口（按 ID、状态、时间等）
- 事务支持
- 数据恢复

**不负责**:
- 业务逻辑
- 状态转换验证

## 三、数据结构重构

### 3.1 统一的 TaskStatus

**位置**: `src/types.ts`

```typescript
/**
 * 任务状态
 *
 * 状态转换图:
 * pending → running → completed
 *              ↓
 *           paused → running
 *              ↓
 *           failed → retrying → running
 *              ↓
 *          cancelled
 */
export type TaskStatus =
  | 'pending'      // 等待执行
  | 'running'      // 执行中
  | 'paused'       // 已暂停
  | 'retrying'     // 重试中
  | 'completed'    // 已完成
  | 'failed'       // 失败
  | 'cancelled';   // 已取消

/**
 * 子任务状态
 */
export type SubTaskStatus =
  | 'pending'      // 等待执行
  | 'running'      // 执行中
  | 'paused'       // 已暂停
  | 'retrying'     // 重试中
  | 'completed'    // 已完成
  | 'failed'       // 失败
  | 'skipped';     // 跳过
```

**删除**:
- ❌ `interrupted` 状态（使用 `cancelled` 替代）
- ❌ 所有其他文件中的 TaskStatus 定义
