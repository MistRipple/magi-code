## 分析结果

### 1. SessionManager 双重存在 - **不是问题**

**结论**: 这**不是**双重系统问题，而是**命名混淆**。

#### 职责完全不同

| 特征 | CLI SessionManager | UnifiedSessionManager |
|------|-------------------|----------------------|
| **职责** | 管理 CLI 进程生命周期 | 管理任务数据持久化 |
| **管理对象** | CLI 进程（claude/codex/gemini） | Session/Task/SubTask 数据 |
| **核心功能** | 启动/停止进程、消息队列、健康监控 | 创建/更新/查询 Session 数据 |
| **持久化** | 不持久化（内存管理） | 持久化到 `.multicli-sessions/` |
| **功能重叠** | 0% | 0% |

#### 与 TaskManager 问题的对比

| 特征 | TaskManager 问题 | SessionManager "问题" |
|------|-----------------|---------------------|
| 功能重叠 | 83% | **0%** |
| 双重调用 | ✅ 存在 | **❌ 不存在** |
| 状态同步 | ✅ 存在 | **❌ 不存在** |
| 职责冲突 | ✅ 冲突 | **❌ 不冲突** |
| 需要迁移 | ✅ 需要 | **❌ 不需要** |

#### 建议
**保持现状**，不需要任何清理。两个 SessionManager 各司其职，架构清晰。

### 2. 状态同步代码残留 - **需要清理**

**结论**: 存在误导性注释和纯日志代码，需要清理。

#### 发现的问题

##### 问题 1: 误导性注释（1 处）
**位置**: `src/orchestrator/orchestrator-agent.ts:3885`

```typescript
/** 将执行计划同步到 UnifiedTaskManager */
private async syncPlanToTaskManager(plan: ExecutionPlan): Promise<void> {
  if (!this.unifiedTaskManager || !this.currentContext) return;

  await this.unifiedTaskManager.updateTask(this.currentContext.taskId, {
    featureContract: plan.featureContract,
    acceptanceCriteria: plan.acceptanceCriteria,
  });

  for (const subTask of plan.subTasks) {
    try {
      await this.unifiedTaskManager.addExistingSubTask(this.currentContext.taskId, subTask);
      this.subTaskIdToTaskIdMap.set(subTask.id, this.currentContext.taskId);
    } catch (error) {
      console.warn('[OrchestratorAgent] 同步子任务失败:', error);
    }
  }

  globalEventBus.emitEvent('task:created', { taskId: this.currentContext.taskId });
}
```

**问题**:
- 注释说"同步到 UnifiedTaskManager"，暗示在两个系统之间同步
- 实际上只是**使用** UnifiedTaskManager，不是同步
- 这是正常的业务逻辑，不是双重系统的状态同步

**应该改为**:
```typescript
/** 将执行计划保存到 UnifiedTaskManager */
```

##### 问题 2: 纯日志事件监听器（5 处）
**位置**: `src/orchestrator/orchestrator-agent.ts:2346-2374`

```typescript
private setupUnifiedTaskManagerEvents(): void {
  if (!this.unifiedTaskManager) return;

  // 监听子任务开始
  this.unifiedTaskManager.on('subtask:started', (task, subTask) => {
    console.log(`[OrchestratorAgent] SubTask started: ${subTask.id}`);
  });

  // 监听子任务完成
  this.unifiedTaskManager.on('subtask:completed', (task, subTask) => {
    console.log(`[OrchestratorAgent] SubTask completed: ${subTask.id}`);
  });

  // 监听子任务失败
  this.unifiedTaskManager.on('subtask:failed', (task, subTask) => {
    console.log(`[OrchestratorAgent] SubTask failed: ${subTask.id}`);
  });

  // 监听子任务重试
  this.unifiedTaskManager.on('subtask:retrying', (task, subTask) => {
    console.log(`[OrchestratorAgent] SubTask retrying: ${subTask.id} (${subTask.retryCount}/${subTask.maxRetries})`);
  });

  // 监听进度更新
  this.unifiedTaskManager.on('subtask:progress', (task, subTask, progress) => {
    console.log(`[OrchestratorAgent] SubTask progress: ${subTask.id} - ${progress}%`);
  });
}
```

**问题**:
- 这些事件监听器**只打印日志**，没有任何业务逻辑
- 不是状态同步代码（之前的状态同步代码已经删除）
- 可能是调试代码残留

**选项**:
1. **删除整个方法**（如果不需要日志）
2. **保留**（如果需要监控日志）
