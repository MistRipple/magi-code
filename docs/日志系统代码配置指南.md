# 日志系统代码配置指南

## 概述

除了环境变量和配置文件，日志系统还提供了丰富的代码级别配置 API，让您可以在运行时动态调整日志行为。

---

## 快速开始

```typescript
import { logger, LogLevel, LogCategory } from './logging';

// 在应用启动时配置
function initializeLogging() {
  // 1. 配置 CLI 消息日志
  logger.configureCLILogging({
    enabled: true,
    logMessages: true,
    logResponses: true,
    maxLength: 500,        // 控制台显示长度
    maxLengthFile: 0       // 文件日志不限制
  });

  // 2. 配置文件日志
  logger.configureFileLogging({
    enabled: true,
    path: '.multicli/logs',
    maxSize: 10 * 1024 * 1024,  // 10MB
    maxFiles: 5
  });

  // 3. 配置控制台输出
  logger.configureConsoleLogging({
    enabled: true,
    colorize: true,
    timestamp: true
  });

  // 4. 设置日志级别
  logger.setLevel(LogLevel.INFO);
  logger.setCategoryLevel(LogCategory.CLI, LogLevel.DEBUG);
}
```

## 配置方法详解

### 1. configureCLILogging - CLI 消息日志配置

配置 CLI 交互消息的日志行为。

```typescript
logger.configureCLILogging({
  enabled?: boolean;          // 是否启用（自动设置 CLI 分类为 DEBUG）
  logMessages?: boolean;      // 是否记录发送的消息
  logResponses?: boolean;     // 是否记录接收的响应
  maxLength?: number;         // 控制台显示的最大长度
  maxLengthFile?: number;     // 文件日志的最大长度（0 = 不限制）
});
```

**示例**：

```typescript
// 启用 CLI 日志，完整保存到文件
logger.configureCLILogging({
  enabled: true,
  logMessages: true,
  logResponses: true,
  maxLength: 500,        // 控制台截断到 500 字符
  maxLengthFile: 0       // 文件保存完整内容
});

// 只记录响应，不记录发送的消息
logger.configureCLILogging({
  logMessages: false,
  logResponses: true
});

// 临时禁用 CLI 日志
logger.configureCLILogging({
  enabled: false
});
```

### 2. configureFileLogging - 文件日志配置

配置文件日志的行为。

```typescript
logger.configureFileLogging({
  enabled: boolean;           // 是否启用文件日志（必需）
  path?: string;              // 日志文件目录
  maxSize?: number;           // 单个文件最大大小（字节）
  maxFiles?: number;          // 保留的文件数量
});
```

**示例**：

```typescript
// 启用文件日志
logger.configureFileLogging({
  enabled: true,
  path: '.multicli/logs',
  maxSize: 10 * 1024 * 1024,  // 10MB
  maxFiles: 5                  // 保留 5 个文件
});

// 更改日志目录
logger.configureFileLogging({
  enabled: true,
  path: '/var/log/multicli'
});

// 禁用文件日志
logger.configureFileLogging({
  enabled: false
});
```

### 3. configureConsoleLogging - 控制台输出配置

配置控制台输出的格式。

```typescript
logger.configureConsoleLogging({
  enabled?: boolean;          // 是否启用控制台输出
  colorize?: boolean;         // 是否使用颜色
  timestamp?: boolean;        // 是否显示时间戳
});
```

**示例**：

```typescript
// 启用彩色输出
logger.configureConsoleLogging({
  enabled: true,
  colorize: true,
  timestamp: true
});

// 禁用颜色（适合日志重定向）
logger.configureConsoleLogging({
  colorize: false
});

// 临时禁用控制台输出
logger.configureConsoleLogging({
  enabled: false
});
```

### 4. setLevel - 设置全局日志级别

```typescript
logger.setLevel(LogLevel.DEBUG);   // 显示所有日志
logger.setLevel(LogLevel.INFO);    // 显示 INFO 及以上
logger.setLevel(LogLevel.WARN);    // 只显示警告和错误
logger.setLevel(LogLevel.ERROR);   // 只显示错误
logger.setLevel(LogLevel.SILENT);  // 禁用所有日志
```

### 5. setCategoryLevel - 设置分类日志级别

```typescript
// 设置特定分类的级别
logger.setCategoryLevel(LogCategory.CLI, LogLevel.DEBUG);
logger.setCategoryLevel(LogCategory.WORKER, LogLevel.INFO);
logger.setCategoryLevel(LogCategory.ORCHESTRATOR, LogLevel.WARN);

// 清除分类级别（回退到全局级别）
logger.clearCategoryLevel(LogCategory.CLI);
```

### 6. updateConfig - 完整配置更新

更新完整的配置对象。

```typescript
logger.updateConfig({
  enabled: true,
  level: LogLevel.INFO,
  categories: {
    [LogCategory.CLI]: LogLevel.DEBUG,
    [LogCategory.WORKER]: LogLevel.INFO,
  },
  console: {
    enabled: true,
    colorize: true,
    timestamp: true,
  },
  file: {
    enabled: true,
    path: '.multicli/logs',
    maxSize: 10 * 1024 * 1024,
    maxFiles: 5,
  },
  cli: {
    logMessages: true,
    logResponses: true,
    maxLength: 500,
    maxLengthFile: 0,
  },
});
```

### 7. getConfig - 获取当前配置

```typescript
const config = logger.getConfig();
console.log('当前日志级别:', LogLevel[config.level]);
console.log('CLI 分类级别:', LogLevel[config.categories[LogCategory.CLI] || config.level]);
console.log('文件日志:', config.file.enabled ? '启用' : '禁用');
```

### 8. resetConfig - 重置为默认配置

```typescript
// 重置为默认配置（会重新加载环境变量）
logger.resetConfig();
```

## 实际应用场景

### 场景 1: 开发环境配置

```typescript
// development.ts
import { logger, LogLevel, LogCategory } from './logging';

export function setupDevelopmentLogging() {
  // 开发环境：显示所有日志
  logger.setLevel(LogLevel.DEBUG);

  // CLI 交互详细日志
  logger.configureCLILogging({
    enabled: true,
    logMessages: true,
    logResponses: true,
    maxLength: 1000,       // 开发环境显示更多内容
    maxLengthFile: 0
  });

  // 启用文件日志
  logger.configureFileLogging({
    enabled: true,
    path: '.multicli/logs/dev',
    maxSize: 50 * 1024 * 1024,  // 50MB
    maxFiles: 10
  });

  // 彩色输出
  logger.configureConsoleLogging({
    enabled: true,
    colorize: true,
    timestamp: true
  });
}
```

### 场景 2: 生产环境配置

```typescript
// production.ts
import { logger, LogLevel, LogCategory } from './logging';

export function setupProductionLogging() {
  // 生产环境：只显示重要日志
  logger.setLevel(LogLevel.INFO);

  // 错误和恢复详细记录
  logger.setCategoryLevel(LogCategory.RECOVERY, LogLevel.DEBUG);

  // CLI 日志只记录到文件
  logger.configureCLILogging({
    enabled: true,
    logMessages: true,
    logResponses: true,
    maxLength: 200,        // 控制台少显示
    maxLengthFile: 0       // 文件完整保存
  });

  // 文件日志必须启用
  logger.configureFileLogging({
    enabled: true,
    path: '/var/log/multicli',
    maxSize: 100 * 1024 * 1024,  // 100MB
    maxFiles: 20
  });

  // 控制台简化输出
  logger.configureConsoleLogging({
    enabled: true,
    colorize: false,       // 生产环境不用颜色
    timestamp: true
  });
}
```

### 场景 3: 测试环境配置

```typescript
// test.ts
import { logger, LogLevel } from './logging';

export function setupTestLogging() {

    enabled: false

  // 禁用文件日志

    enabled: true,
    colorize: false,
    timestamp: false
  });
}
```

### 场景 4: 调试特定问题

```typescript
// 临时启用详细日志调试 CLI 问题
function debugCLIIssue() {
  // 保存当前配置
  const originalConfig = logger.getConfig();

  try {
    // 启用详细 CLI 日志
    logger.setCategoryLevel(LogCategory.CLI, LogLevel.DEBUG);
    logger.configureCLILogging({
      enabled: true,
      logMessages: true,
      logResponses: true,
      maxLength: 5000,       // 显示更多内容
      maxLengthFile: 0
    });

    // 执行调试操作
    // ...

  } finally {
    // 恢复原配置
    logger.updateConfig(originalConfig);
  }
}
```

### 场景 5: 动态调整日志级别

```typescript
// 根据用户设置动态调整
function applyUserLogSettings(userSettings: {
  verbosity: 'quiet' | 'normal' | 'verbose' | 'debug';
  saveLogs: boolean;
}) {
  // 根据详细程度设置级别
  const levelMap = {
    quiet: LogLevel.ERROR,
    normal: LogLevel.INFO,
    verbose: LogLevel.INFO,
    debug: LogLevel.DEBUG,
  };

  logger.setLevel(levelMap[userSettings.verbosity]);

  // 调试模式下启用所有分类的详细日志
  if (userSettings.verbosity === 'debug') {
    Object.values(LogCategory).forEach(category => {
      logger.setCategoryLevel(category, LogLevel.DEBUG);
    });
  }

  // 根据用户设置启用/禁用文件日志
  logger.configureFileLogging({
    enabled: userSettings.saveLogs,
    path: '.multicli/logs',
    maxSize: 10 * 1024 * 1024,
    maxFiles: 5
  });
}
```

### 场景 6: 性能敏感场景

```typescript
// 在性能敏感的操作中临时禁用日志
async function performanceIntensiveOperation() {
  // 保存当前配置
  const wasEnabled = logger.getConfig().enabled;

  try {
    // 临时禁用日志
    logger.updateConfig({ enabled: false });

    // 执行性能敏感操作
    await heavyComputation();

  } finally {
    // 恢复日志
    logger.updateConfig({ enabled: wasEnabled });
  }
}
```

## 配置优先级

配置的优先级从高到低：

1. **代码配置** - `logger.setLevel()`, `logger.configureCLILogging()` 等
2. **环境变量** - `MULTICLI_LOG_LEVEL`, `MULTICLI_LOG_CLI` 等
3. **配置文件** - `.multicli/logging.json`
4. **默认配置** - 内置的默认值

**示例**：

```typescript
// 1. 默认配置：INFO
// 2. 环境变量：MULTICLI_LOG_LEVEL=WARN
// 3. 代码配置：logger.setLevel(LogLevel.DEBUG)
// 最终结果：DEBUG（代码配置优先级最高）
```

## 配置持久化

代码配置是运行时的，不会持久化。如果需要持久化配置：

### 方法 1: 保存到配置文件

```typescript
import * as fs from 'fs';
import * as path from 'path';

function saveLogConfig() {
  const config = logger.getConfig();
  const configPath = path.join(process.cwd(), '.multicli', 'logging.json');

  fs.mkdirSync(path.dirname(configPath), { recursive: true });
  fs.writeFileSync(configPath, JSON.stringify(config, null, 2));
}
```

### 方法 2: 在启动时加载

```typescript
// app.ts
import { setupLogging } from './config/logging';

async function main() {
  // 在应用启动时配置日志
  setupLogging();

  // 其他初始化
  // ...
}
```

## 最佳实践

### 1. 集中配置

```typescript
// config/logging.ts
import { logger, LogLevel, LogCategory } from '../logging';

export function setupLogging(env: 'development' | 'production' | 'test') {
  const configs = {
    development: () => {
      logger.setLevel(LogLevel.DEBUG);
      logger.configureCLILogging({
        enabled: true,
        maxLength: 1000,
        maxLengthFile: 0
      });
      logger.configureFileLogging({
        enabled: true,
        path: '.multicli/logs/dev'
      });
    },
    production: () => {
      logger.setLevel(LogLevel.INFO);
      logger.configureCLILogging({
        enabled: true,
        maxLength: 200,
        maxLengthFile: 0
      });
      logger.configureFileLogging({
        enabled: true,
        path: '/var/log/multicli'
      });
    },
    test: () => {
      logger.setLevel(LogLevel.ERROR);
      logger.configureCLILogging({ enabled: false });
      logger.configureFileLogging({ enabled: false });
    },
  };

  configs[env]();
}
```

### 2. 环境检测

```typescript
function setupLogging() {
  const env = process.env.NODE_ENV || 'development';

  if (env === 'production') {
    setupProductionLogging();
  } else if (env === 'test') {
    setupTestLogging();
  } else {
    setupDevelopmentLogging();
  }
}
```

### 3. 配置验证

```typescript
function validateLogConfig() {
  const config = logger.getConfig();

  if (config.file.enabled && !config.file.path) {
    throw new Error('文件日志已启用但未指定路径');
  }

  if (config.file.maxSize < 1024 * 1024) {
    console.warn('文件大小限制过小，建议至少 1MB');
  }
}
```

## 调试技巧

### 查看当前配置

```typescript
function printLogConfig() {
  const config = logger.getConfig();

  console.log('=== 日志配置 ===');
  console.log('全局级别:', LogLevel[config.level]);
  console.log('分类级别:');
  Object.entries(config.categories).forEach(([cat, level]) => {
    console.log(`  ${cat}: ${LogLevel[level]}`);
  });
  console.log('控制台:', config.console.enabled ? '启用' : '禁用');
  console.log('文件日志:', config.file.enabled ? '启用' : '禁用');
  console.log('CLI 日志:', config.cli.logMessages || config.cli.logResponses ? '启用' : '禁用');
}
```

## 总结

代码配置提供了最灵活的日志控制方式：

✅ **运行时动态调整** - 无需重启应用
✅ **细粒度控制** - 精确控制每个方面
✅ **环境适配** - 根据环境自动配置
✅ **调试友好** - 快速启用/禁用特定日志

**推荐做法**：
1. 使用代码配置设置基础配置
2. 使用环境变量覆盖特定设置
3. 在调试时临时调整配置
4. 生产环境使用保守的配置
