**位置**: `src/orchestrator/orchestrator-agent.ts:2298-2305`

```typescript
const taskId = this.currentContext?.taskId || '';
for (const task of tasks) {
  // 调用 TaskManager (旧)
  this.taskManager?.updateSubTaskStatus(taskId, task.id, 'running');

  // 调用 UnifiedTaskManager (新)
  if (this.unifiedTaskManager && taskId) {
    const mappedTaskId = this.subTaskIdToTaskIdMap.get(task.id) || taskId;
    this.unifiedTaskManager.startSubTask(mappedTaskId, task.id).catch(error => {

    });
  }
}
```

**问题**: 同一个操作调用了两个管理器

### 示例 2: 事件监听中的状态同步

**位置**: `src/orchestrator/orchestrator-agent.ts:2357-2390`

```typescript
// 监听子任务开始
this.unifiedTaskManager.on('subtask:started', (task, subTask) => {
  console.log(`[OrchestratorAgent] SubTask started: ${subTask.id}`);
  // 同步到 TaskManager (old)
  if (this.taskManager) {
    this.taskManager.updateSubTaskStatus(task.id, subTask.id, 'running');
  }
});

// 监听子任务完成
this.unifiedTaskManager.on('subtask:completed', (task, subTask) => {
  console.log(`[OrchestratorAgent] SubTask completed: ${subTask.id}`);
  if (this.taskManager) {
    this.taskManager.updateSubTaskStatus(task.id, subTask.id, 'completed');
  }
});

// 监听子任务失败
this.unifiedTaskManager.on('subtask:failed', (task, subTask) => {
  console.log(`[OrchestratorAgent] SubTask failed: ${subTask.id}`);
  if (this.taskManager) {
    this.taskManager.updateSubTaskStatus(task.id, subTask.id, 'failed');
  }
});

// 监听子任务重试
this.unifiedTaskManager.on('subtask:retrying', (task, subTask) => {
  console.log(`[OrchestratorAgent] SubTask retrying: ${subTask.id}`);
  if (this.taskManager) {
    this.taskManager.updateSubTaskStatus(task.id, subTask.id, 'running');
  }
});
```

**问题**: 这是典型的状态同步代码，和之前的 TaskStateManager 问题一模一样！

## 🎯 问题根源

### 为什么会有两个管理器？

1. **TaskManager**:
   - 原始的任务管理器
   - 直接操作 UnifiedSessionManager
   - 用于 UI 展示和会话管理

2. **UnifiedTaskManager**:
   - 新的统一任务管理器
   - 支持高级特性（优先级、超时、重试）
   - 用于任务执行和状态追踪

### 为什么会双重调用？

**原因**: 在引入 UnifiedTaskManager 时，没有完全替换 TaskManager，而是采用了"双重调用"的方式来保持兼容性。

**类似问题**: 这和之前的 TaskStateManager + UnifiedTaskManager 问题完全一样！
