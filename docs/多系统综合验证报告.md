### 2.3 已知问题

#### P0 - 双重状态管理系统

**问题描述**:
- TaskManager 和 TaskStateManager 职责重叠
- 可能导致状态不一致
- 开发者困惑何时使用哪个

**影响**: 状态可能不同步，维护成本高

**建议**:
- 方案 A: 合并为一个 TaskManager
- 方案 B: 明确职责分离并文档化

#### P1 - TaskStatus 类型定义冲突

**问题描述**:
```typescript
// types.ts
type TaskStatus = 'pending' | 'running' | 'interrupted' |
                  'completed' | 'failed' | 'cancelled';

// task-state-manager.ts
type TaskStatus = 'pending' | 'running' | 'completed' |
                  'failed' | 'retrying' | 'cancelled';
```

**影响**: 类型不一致导致编译错误或运行时错误

**建议**: 统一使用 types.ts 中的定义，添加 'retrying' 状态

#### P2 - TaskDependencyGraph 集成不清晰

**问题描述**:
- TaskDependencyGraph 实现完善，但使用不明确
- 没有在 OrchestratorAgent 中看到明确集成

**影响**: 依赖关系可能未被正确处理

**建议**: 在 OrchestratorAgent 中明确使用 TaskDependencyGraph

## 三、文件快照系统（Snapshot System）

### 验证状态: ✅ P0 问题已修复

**修复日期**: 2026-01-18
**测试状态**: 4/4 测试通过

### 3.1 设计意图验证 ✅ 合理

**核心设计**:
```
任务前: 创建快照保存初始状态
  ↓
并行任务执行 (可能同时修改同一文件)
  ↓
FileLockManager 保证写入顺序
  ↓
用户自主选择何时确认变更
  ↓
确认后: 删除旧快照 + 立即创建新基准快照
```

**并发控制**: 双重保护
```
1. TaskDependencyGraph (静态): 调度前检测文件冲突，串行化调度
2. FileLockManager (动态): 运行时加锁，保证写入顺序
```

### 3.2 已修复的 P0 问题

#### 问题 1: 快照创建时机错误 ✅ 已修复

**原问题**:
- 在 executeCLI() **之后**创建快照
- 保存的是修改后的内容，而非初始状态
- 回滚失效

**修复位置**: [src/orchestrator/worker-agent.ts:272-284](src/orchestrator/worker-agent.ts#L272-L284)

**修复方案**:
```typescript
// ✅ 修复: 事前创建快照

  for (const file of subTask.targetFiles) {
    this.snapshotManager.createSnapshot(file, this.type, subTask.id);
  }
}

// 然后执行任务
const response = await this.executeCLI(prompt);
```

**验证**: scripts/test-snapshot-fix.js 测试通过

#### 问题 2: 确认后未创建新快照 ✅ 已修复

**原问题**:
- acceptChange() 只删除旧快照

- 后续任务无法回滚到确认后的状态

**修复方案**:

### 5.1 完整数据流图

```
用户输入
  ↓

  ↓
TaskAnalyzer.analyze()
  ├─ ProfileLoader.getCategory() → riskLevel, recommendedWorker
  └─ 返回 TaskAnalysis
  ↓
buildPlanFromAnalysis()
  ├─ TaskSplitter.split()
  │   └─ CLISelector.select(analysis, recommendedWorker)
  │       ├─ ProfileLoader (画像推荐)
  │       ├─ ExecutionStats (统计推荐)
  │       └─ ConflictResolver (冲突解决)
  └─ 生成 ExecutionPlan (包含 riskLevel)
  ↓
WorkerPool.dispatch()
  ├─ TaskDependencyGraph.analyze()
  │   ├─ 拓扑排序
  │   ├─ 文件冲突检测
  │   └─ 生成 ExecutionBatch[]
  ├─ FileLockManager.acquire()
  └─ WorkerAgent.executeSubTask()
      ├─ SnapshotManager.createSnapshot() (事前)
      ├─ executeCLI()
      └─ 返回 ExecutionResult
  ↓
ContextManager
  ├─ addMessage() (即时上下文)
  ├─ 自动迁移到 Memory (超过轮数限制)
  └─ ContextCompressor.compress() (需要时)
  ↓
UnifiedSessionManager
  └─ 持久化所有状态
```
