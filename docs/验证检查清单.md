### 5. 完整性检查脚本
```bash
#!/bin/bash

echo "=== 快速验证检查 ==="
echo ""

# 1. 编译
echo "1. 编译检查..."
if npx tsc --noEmit 2>&1 | grep -q "error TS"; then
  echo "❌ 编译失败"
  exit 1
else
  echo "✅ 编译通过"
fi

# 2. 测试
echo ""
echo "2. 测试检查..."
if npm test 2>&1 | grep -q "✅ 通过: 33/33"; then
  echo "✅ 测试通过 (33/33)"
else
  echo "❌ 测试失败"
  exit 1
fi

# 3. TaskStateManager 使用
echo ""
echo "3. TaskStateManager 清理检查..."
count=$(grep -c "this.taskStateManager" src/orchestrator/orchestrator-agent.ts 2>/dev/null || echo "0")
if [ "$count" -eq 0 ]; then
  echo "✅ 无 TaskStateManager 引用"
else
  echo "❌ 发现 $count 个 TaskStateManager 引用"
  exit 1
fi

# 4. RecoveryHandler
echo ""
echo "4. RecoveryHandler 检查..."
if grep -A 5 "new RecoveryHandler" src/orchestrator/orchestrator-agent.ts | grep -q "unifiedTaskManager"; then
  echo "✅ RecoveryHandler 使用 UnifiedTaskManager"
else
  echo "❌ RecoveryHandler 未使用 UnifiedTaskManager"
  exit 1
fi

echo ""
echo "=== ✅ 所有检查通过 ==="
```

保存为 `scripts/verify-migration.sh` 并运行：
```bash
chmod +x scripts/verify-migration.sh
./scripts/verify-migration.sh
```

## 问题排查指南

### 问题 1: 编译错误

**症状**: `npx tsc --noEmit` 报错

**排查步骤**:
1. 检查错误信息中的文件和行号
2. 确认类型定义是否正确
3. 检查导入路径是否正确
4. 清理缓存: `rm -rf node_modules/.cache`

### 问题 2: 测试失败

**症状**: `npm test` 失败

**排查步骤**:
1. 查看失败的测试用例
2. 检查测试数据是否正确
3. 确认功能是否正常工作
4. 运行单个测试: `npm test -- test-name`

### 问题 3: TaskStateManager 引用

**症状**: 发现 `this.taskStateManager` 引用

**排查步骤**:
1. 确认引用位置
2. 替换为 `this.unifiedTaskManager`
3. 更新方法调用
4. 运行测试确认

### 问题 4: RecoveryHandler 错误

**症状**: RecoveryHandler 初始化失败

**排查步骤**:
1. 检查构造函数参数
2. 确认传入 UnifiedTaskManager
3. 检查 UnifiedTaskManager 是否已初始化
4. 查看错误日志
