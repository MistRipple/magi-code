# 统一日志管理系统设计

**设计时间**: 2026-01-18 16:45
**目标**: 提供统一的日志管理，方便开发调试，特别是 CLI 交互的消息检查和内容展示

---

## 需求分析

### 当前问题

1. **日志分散**: 584 处 `console.log/warn/error` 分散在各个文件
2. **无统一控制**: 无法通过配置统一开关日志
3. **调试困难**: CLI 交互消息难以追踪和查看
4. **缺乏结构**: 日志格式不统一，难以过滤和分析
5. **已有基础**: 存在 `MessageTracer` 但功能有限

### 用户需求

1. ✅ **统一日志管理**: 所有日志通过统一接口输出
2. ✅ **系统配置开关**: 可通过配置文件控制日志级别
3. ✅ **CLI 交互可视化**: 清晰展示 CLI 消息的发送和接收
4. ✅ **内容格式化**: 自动格式化和高亮显示
5. ✅ **过滤和搜索**: 支持按模块、级别、关键词过滤

---

## 架构设计

### 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层                                    │
│  (orchestrator, worker-pool, cli-adapter, etc.)                 │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    UnifiedLogger (统一日志接口)                  │
│  - debug(), info(), warn(), error()                             │
│  - logCLIMessage(), logCLIResponse()                            │
│  - logTaskEvent(), logWorkerEvent()                             │
└────────────────────────────┬────────────────────────────────────┘
                             │
                ┌────────────┼────────────┐
                ▼            ▼            ▼
        ┌──────────┐  ┌──────────┐  ┌──────────┐
        │ Console  │  │  File    │  │ Message  │
        │ Handler  │  │ Handler  │  │ Tracer   │
        └──────────┘  └──────────┘  └──────────┘
```

### 核心组件

#### 1. UnifiedLogger (统一日志器)
- 提供统一的日志接口
- 支持多种日志级别
- 支持模块化日志
- 集成 MessageTracer

#### 2. LogHandler (日志处理器)
- ConsoleHandler: 控制台输出（带颜色和格式化）
- FileHandler: 文件输出（持久化日志）
- TracerHandler: 集成 MessageTracer

#### 3. LogConfig (日志配置)
- 从配置文件读取
- 支持环境变量覆盖
- 支持运行时修改

## 详细设计

### 1. 日志级别

```typescript
enum LogLevel {
  DEBUG = 0,    // 详细调试信息
  INFO = 1,     // 一般信息
  WARN = 2,     // 警告信息
  ERROR = 3,    // 错误信息
  SILENT = 4,   // 静默（不输出）
}
```

### 2. 日志分类

```typescript
enum LogCategory {
  SYSTEM = 'system',           // 系统级日志
  CLI = 'cli',                 // CLI 交互日志
  TASK = 'task',               // 任务管理日志
  WORKER = 'worker',           // Worker 执行日志
  ORCHESTRATOR = 'orchestrator', // 编排器日志
  SESSION = 'session',         // 会话管理日志
  RECOVERY = 'recovery',       // 恢复机制日志
  UI = 'ui',                   // UI 交互日志
}
```

### 3. CLI 消息日志格式

```typescript
interface CLIMessageLog {
  timestamp: number;
  direction: 'send' | 'receive';
  cli: CLIType;
  role: 'worker' | 'orchestrator';
  requestId: string;

  // 消息内容
  message: {
    raw: string;           // 原始消息
    formatted: string;     // 格式化后的消息
    truncated: boolean;    // 是否被截断
    length: number;        // 原始长度
  };

  // 响应内容（仅 receive）
  response?: {
    raw: string;
    formatted: string;
    truncated: boolean;
    length: number;
    duration: number;      // 响应时间（ms）
  };

  // 元数据
  metadata?: {
    taskId?: string;
    subTaskId?: string;
    intent?: string;
  };
}
```

### 4. 配置文件格式

```json
{
  "logging": {
    "enabled": true,
    "level": "info",
    "categories": {
      "system": "info",
      "cli": "debug",
      "task": "info",
      "worker": "debug",
      "orchestrator": "debug",
      "session": "warn",
      "recovery": "info",
      "ui": "info"
    },
    "handlers": {
      "console": {
        "enabled": true,
        "colorize": true,
        "format": "pretty"
      },
      "file": {
        "enabled": false,
        "path": ".multicli-logs/",
        "maxSize": "10MB",
        "maxFiles": 5
      },
      "tracer": {
        "enabled": true,
        "includePayload": false,
        "maxRecords": 1000
      }
    },
    "cli": {
      "logMessages": true,
      "logResponses": true,
      "maxMessageLength": 500,
      "maxResponseLength": 1000,
      "highlightKeywords": ["error", "warning", "success"]
    }
  }
}
```

### 5. 环境变量覆盖

```bash
# 全局日志级别
MULTICLI_LOG_LEVEL=debug

# 特定分类日志级别
MULTICLI_LOG_CLI=debug
MULTICLI_LOG_TASK=info

# 启用/禁用日志
MULTICLI_LOG_ENABLED=true

# CLI 消息日志
MULTICLI_LOG_CLI_MESSAGES=true
MULTICLI_LOG_CLI_RESPONSES=true
```

## 使用示例

### 基本日志

```typescript
import { logger } from './logging';

// 基本日志
logger.debug('调试信息');
logger.info('一般信息');
logger.warn('警告信息');
logger.error('错误信息');

// 带分类的日志
logger.info('任务已创建', { category: 'task', taskId: 'task-123' });
logger.debug('Worker 开始执行', { category: 'worker', workerId: 'worker-1' });
```

### CLI 消息日志

```typescript
// 发送消息到 CLI
logger.logCLIMessage({
  cli: 'claude',
  role: 'worker',
  requestId: 'req-123',
  message: 'Please implement the feature...',
  metadata: { taskId: 'task-123', subTaskId: 'subtask-1' }
});

// 接收 CLI 响应
logger.logCLIResponse({
  cli: 'claude',
  role: 'worker',
  requestId: 'req-123',
  response: 'I have implemented the feature...',
  duration: 5000,
  metadata: { taskId: 'task-123', subTaskId: 'subtask-1' }
});
```

## 性能考虑

### 1. 延迟初始化
- 只在需要时创建日志处理器
- 避免不必要的字符串格式化

### 2. 异步写入
- 文件日志使用异步写入
- 避免阻塞主线程

### 3. 消息截断
- 自动截断过长的消息
- 可配置最大长度

### 4. 条件日志
```typescript
// 只在 DEBUG 级别时执行昂贵的操作
if (logger.isDebugEnabled()) {
  logger.debug('详细信息', expensiveOperation());
}
```
