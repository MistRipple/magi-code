如果你看到 CORS 错误：
  "Access to script ... has been blocked by CORS policy"

这说明你在 **浏览器** 中直接打开了 HTML 文件。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ 错误的测试方法：
   - 双击打开 index.html
   - 在浏览器中打开文件
   - 使用 file:// 协议

✅ 正确的测试方法：
   1. 在 VSCode 中打开 MultiCLI 项目
   2. 按 F5 启动扩展开发主机
   3. 在新窗口中打开 MultiCLI 面板

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

为什么必须在 VSCode 中测试？

1. Webview URI 转换
   我们的修复使用 webview.asWebviewUri() 将路径转换为
   vscode-webview:// 协议，这只在 VSCode Webview 中有效。

2. Import Map
   我们生成的 Import Map 使用 vscode-webview:// 协议，
   浏览器的 file:// 协议无法处理。

3. Content Security Policy
   VSCode Webview 有特殊的 CSP 配置。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

快速开始：

  1. 按 F5 启动扩展开发主机
  2. 新窗口标题应显示 "[Extension Development Host]"
  3. 打开 MultiCLI 面板
  4. 检查样式是否正确加载
